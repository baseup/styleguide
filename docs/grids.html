Grid System

How it works

Our grid system is composed of a 12 flexible columns with a 30px gutter in-between columns. It uses a series of containers, rows, and columns to layout and align content. It’s built with flexbox and is fully responsive.

Here's an example at how the grid comes together:



Each grid columns have a 15px padding on each side that results to a 30px gutter between columns and a 15px gutter on the edges.



Responsive Columns

The grid system includes five tiers of predefined classes for building complex responsive layouts. Customize the size of your columns on extra small, small, medium, large, or extra large devices however you see fit.

  Class prefix Viewport width
Extra small .col- <576px
Small .col-sm- ≥576px
Medium .col-md- ≥768px
Large .col-lg- ≥992px
Extra Large .col-xl- ≥1200px

All breakpoints

For grids that are the same from the smallest of devices to the largest, use the .col and .col-* classes. Specify a numbered class when you need a particularly sized column; otherwise, just specify .col and it fill in the remaining space.


Mix and match

Use a combination of different classes for each tier as needed. See the example below for a better idea of how it all works.


Nesting

To nest your content with the default grid, add a new .row and set of .col-sm-* columns within an existing .col-sm-* column. Nested rows should include a set of columns that add up to 12 or fewer (it is not required that you use all 12 available columns).


No gutters

The gutters between columns in our predefined grid classes can be removed with .no-gutters. This removes the negative margins from .row and the horizontal padding from all immediate children columns.
